HCI Spring 2021 - Project 2: Car Dashboard Design

1. Motivation


Design is the key aspect in communicating between objects and users. Any device should be intuitive and easy to use. That said, some give the user a tough time figuring out how they work. In this project, we will take a closer look into the car, a ubiquitous mean of transport we use every day, to see how its dashboard is designed. Then, we will propose a new design to make it more user-friendly.

3. Features

To see if the interface makes doing those common things easier, first, we need to identify common uses and rare uses of the dashboard.


Common uses:

  • Warning light: Dashboard shows crutial warnings of the current situation to inform the driver. For example, if the engine has a serious problem, the "check engine" light will be shown up. Another example, when the "fuel" turned on, it remind the drivers to refill the tank before their cars running out of gas.

  • Gauges: Well known gauges are speedometer, tachometer, temperature and gas gauges. It will inform you how fast you're going (the speedometer) so that we can adjust the speed properly. Another important gauge is the gas gauge which informs you how much gas you have left.

Rare uses

  • The tachometer, which tells how many times the engine makes one full rotation every minute (rpm). It is not really used a lot.

  • The temperature gauge indicates the current temperature of the engine, which allows the driver to know if it's over/under heating or not. However, it is not necessary to see the temperature all the time. We can use a icon to show if it's over/under heating instead.

4. Potential Issues

What are the potential issues with this design? Too many gauges, most of them do not need to be "always shown" to the driver. It may make drivers distract as they can not focus on essential features.



5. Suggestions

It should easy to focus on the primary features (XYZ), we can remove some rarely-used gauges until the driver needs to know. For example, we can remove the temperature gauge, and only inform the driver when something goes wrong with the temperature, like show up an light icon and sound effecta. For some important information needing to be displayed, we should re-organize them.
